#### 高并发处理的思路及手段
``` 
1.扩容
2.缓存Cache
3.消息队列MQ
```

- 1.高并发之扩容思路
``` 
垂直扩容：升级内存等性能 
水平扩容：增加服务器节点 集群 分片 网络连接开销

数据库是典型共享资源，扩容：
  读多? 写多?
  读扩展：memcache,redis,CND等缓存
  写扩展：hbase,cassandra等
```

- 2.高并发之缓存
```
缓存非常适合读多写少的场景
命中率：命中数/(命中数+没有命中数)
最大元素(空间)
清空策略：缓存空间不够时的清空策略
  FIFO：先进先出 根据时间
  LFU：清除使用次数最少的数据 根据命中率
  LRU：清楚最近一次使用时间最早的数据 清空非热点数据 【最常用】
  过期时间
  随机

本地缓存：编程实现(成员变量、局部变量、静态变量)、guava cache
分布式缓存：memcache,redis
  
guava cache灵感来源于currenthashmap  
memcache：客户端做路由(一致性hash)
  内存空间分为slab_class slab page chunk
redis：分布式
  支持持久化
  支持数据备份 master-slave
  读写性能极高
  
高并发下缓存场景问题：
  缓存一致性
  缓存并发问题
  缓存穿透问题
  缓存的雪崩现象
```
